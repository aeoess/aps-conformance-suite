# Native action_ref canonicalization fixtures

The APS-native action_ref is SHA-256 over the RFC 8785 canonicalization of
{agentId, actionType, scopeRequired, timestamp}. Before canonicalization,
each scope string is normalized to NFC and the scopeRequired array is sorted
by Unicode code point (draft-pidlisnyi-aps-03 section 4.1). Code-point order
differs from UTF-16 code-unit order for astral-plane characters; the
astral-scope vector pins that difference.

Four vectors: plain single-scope ASCII, unsorted multi-scope ASCII, an NFD
scope that normalizes to NFC, and an astral-plane scope ordering after a
U+E000..U+FFFF scope. Generated by the agent-passport-system TypeScript
reference and cross-verified byte-identically by the Go implementation.

These vectors carry expected action_ref values rather than the
canonical-bytes contract fields. Both suite runners recompute action_ref for
each input, NFC-normalizing every scope string, sorting scopeRequired by
Unicode code point, then JCS + SHA-256, and assert the recomputed hex and
canonical scope order match the recorded values (TS: computeNativeActionRef in
runners/ts/verify.ts; Go: checkActionRefVectors in runners/go/main.go via the
agent-passport-go actionref package). External implementations verify the same
way.
