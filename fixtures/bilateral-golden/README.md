# Bilateral golden vector: aud + action_ref in the signed body

One BilateralReceipt whose signed body carries both an audience binding
(aps:audience-binding:v1) and the native action_ref. Two vectors, one per
co-signature, each carrying the full canonicalization-and-signature
contract: input (the signable body), canonical_bytes_hex, canonical_sha256,
and an Ed25519 signature that verifies over the canonical bytes.

Provenance: independently derived and cross-verified. The receipt was
generated by the agent-passport-system TypeScript reference
(createBilateralReceipt) and the signable bytes were re-derived from scratch
by an SDK-free Python RFC 8785 implementation (the gateway repo's
tools/verify-bilateral-independent.py), byte-identical by sha256, with both
signatures verified. The signable body deliberately contains no null values
and no numbers, the domain on which strict RFC 8785 and the SDK's
canonicalize agree exactly.

Test keys only; never production key material.
